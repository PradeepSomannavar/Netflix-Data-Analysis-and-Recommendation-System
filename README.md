# Netflix Hybrid Recommendation System using NLP and Similarity Learning

## Project Introduction

Content discovery remains one of the most critical challenges for modern OTT (Over-the-Top) streaming platforms. With thousands of titles available at any given time, users frequently struggle with decision fatigue, and platforms risk high churn when viewers fail to find relevant content. Effective personalization is therefore central to engagement and retention strategies.

This project is fundamentally an **EDA-first, insight-driven data science project**. It begins with extensive **Exploratory Data Analysis (EDA)** of the Netflix catalog — uncovering content distribution patterns, temporal trends, genre structures, and creator influence — and uses those findings to directly inform every subsequent engineering and modeling decision. The recommendation system is an advanced extension built on top of the understanding derived from that analysis.

The system engineers a **hybrid content-based recommendation engine** inspired by Netflix's own recommendation architecture. It combines **Natural Language Processing (NLP)**, **structured feature engineering**, and **similarity learning** to surface contextually relevant titles. Crucially, the selection and weighting of every feature is grounded in what the EDA revealed about the data — not assumed a priori.

Unlike basic content-based filtering that relies solely on genre labels or one-dimensional metadata, this system constructs a multi-dimensional content representation that integrates both unstructured text signals and structured numerical attributes — producing recommendations that are more precise, interpretable, and resistant to noise.

---

## Project Overview

The project follows an **analysis-first methodology**, where EDA informs the design of the machine learning system. The end-to-end pipeline is structured as follows:

```
Data Ingestion
    → Exploratory Data Analysis (EDA) & Insight Extraction
    → Data Cleaning & Preprocessing
    → Feature Engineering (EDA-informed, Text + Numerical)
    → TF-IDF Vectorization
    → Hybrid Similarity Computation
    → Recommendation Generation
    → Interactive Widget-Based UI
```

Recommendations are generated by computing a **hybrid similarity score** that weighs both textual metadata (genres, directors, countries, ratings) and quantitative attributes (duration, release age, content type). The specific fields selected and their respective weights were determined by the statistical significance and distribution patterns observed during EDA.

## Dataset Description

The dataset is a publicly available Netflix catalog snapshot containing metadata for **Movies and TV Shows** available on the platform. It includes the following key columns:

| Column | Description |
|---|---|
| `title` | Title of the content |
| `type` | Movie or TV Show |
| `director` | Director(s) of the content |
| `country` | Country of origin |
| `listed_in` | Genre tags (multi-label) |
| `rating` | Content maturity rating (e.g., TV-MA, PG-13) |
| `duration` | Runtime in minutes or number of seasons |
| `release_year` | Year the title was originally released |
| `date_added` | Date the title was added to Netflix |
| `description` | Short synopsis |

The dataset contained significant missing values, inconsistent formats, and unstructured text fields that required extensive preprocessing before any modeling could take place.

## Exploratory Data Analysis (EDA) & Insights

Before any feature engineering or modeling, the dataset was subjected to rigorous exploratory analysis. The objective was to understand the structure, distributions, and relationships within the data — and to derive decisions that would directly shape the recommendation system's design.

### Content Type Distribution
- **Analysis:** A count plot and pie chart were used to visualize the ratio of Movies to TV Shows in the catalog.
- **Visualization:** Bar chart (Seaborn `countplot`) and pie chart (Matplotlib `pie`).
- **Insight:** Movies significantly outnumber TV Shows. This distribution informed the creation of the `is_movie` binary feature, allowing the model to separate content types in the numeric similarity space.

### Temporal Trends — Yearly and Monthly Additions
- **Analysis:** Line plots were used to track how content additions changed year-over-year and month-over-month, separated by content type.
- **Visualization:** Matplotlib line plots on `year_added` and `month_added` fields extracted from `date_added`.
- **Insight:** The catalog grew substantially after 2015, with seasonal spikes in Q4. This justified engineering `release_age` as a numeric feature — older content behaves differently in user preference patterns than recently added titles.

### Genre Popularity and Diversity
- **Analysis:** Genre tags in `listed_in` were split, exploded, and counted across the full catalog to identify the most prevalent content categories.
- **Visualization:** Horizontal bar chart (Seaborn `barplot`) of the top 10 genres by frequency.
- **Insight:** A small set of genres (International Movies, Dramas, Comedies) dominates the catalog. Genre concentration confirmed that `listed_in` carries the highest discriminative signal for content similarity — directly motivating its **3x weight** in the TF-IDF feature vector.

### Country-Wise Content Distribution
- **Analysis:** Content was aggregated by `country` to identify the top producing nations.
- **Visualization:** Bar chart of the top 10 countries by title count.
- **Insight:** The United States, India, and the United Kingdom dominate content production, but a significant long tail of countries exists. This reinforced the inclusion of `country` as a soft thematic signal in the text features, capturing regional and cultural content flavor.

### Rating Distribution
- **Analysis:** Maturity ratings were counted and ordered by frequency.
- **Visualization:** Bar chart and pie chart (top 8 ratings) using Matplotlib.
- **Insight:** TV-MA and TV-14 dominate the catalog. The `rating` field was included in the text features to capture target audience alignment — content with the same rating tends to share tonal and thematic characteristics.

### Director Frequency Analysis
- **Analysis:** Director contribution was measured by title count to identify the most prolific creators on the platform.
- **Visualization:** Bar chart of the top 10–15 directors by frequency (Matplotlib), and a word cloud of all movie titles (WordCloud library). The word cloud revealed dominant title vocabulary and thematic clusters across the movie catalog.
- **Insight:** A small number of directors are responsible for disproportionately large portions of the catalog. Stylistic fingerprints are strong signals for content affinity — this directly motivated the **2x weight** applied to the `director` field in the TF-IDF feature vector.

### EDA-to-Modeling Connection

Every modeling decision is traceable to a specific EDA finding:

| EDA Finding | Modeling Decision |
|---|---|
| Genres dominate stylistic identity | `listed_in` weighted 3x in TF-IDF |
| Directors signal strong stylistic affinity | `director` weighted 2x in TF-IDF |
| Movies vs TV Shows differ structurally | `is_movie` binary flag as numeric feature |
| Content age affects relevance | `release_age` as numeric feature |
| Country reflects cultural flavor | `country` included as soft text signal |
| Rating reflects target audience | `rating` included in text features |

## Data Cleaning & Preprocessing

- **Missing Values:** Rows with missing `director` and `country` fields were removed, as these are core features for the similarity model. Other fields such as `rating` and `listed_in` were filled with placeholder values to avoid null propagation.
- **Duplicate Removal:** Exact duplicate records were dropped to prevent inflated similarity scores.
- **Duration Parsing:** The `duration` column contained mixed formats (`90 min`, `3 Seasons`). A regex-based extraction function was implemented to parse integer minute values exclusively for movies. TV show durations were treated separately.
- **Date-Time Feature Extraction:** The `date_added` field was parsed into `year_added` and `month_added` for temporal analysis and trend visualization.
- **Text Normalization:** Director names, genres, and country strings were normalized by removing whitespace and converting to lowercase. This ensures that multi-token entities (e.g., `"David Fincher"` becomes `davidfincher`) are treated as single, semantically meaningful tokens by the TF-IDF vectorizer — preventing false matches on common substrings.

## Feature Engineering

All features were selected and weighted based on findings from the EDA phase. No feature was included arbitrarily — each has a traceable motivation from the data analysis.

### Text Features

The following text fields were combined into a single composite feature string used for NLP vectorization:

- **`title`** — The content title (normalized to lowercase)
- **`listed_in`** (Genres) — Weighted 3x; EDA confirmed genres are the single most discriminative signal for grouping similar content
- **`director`** — Weighted 2x; director frequency analysis revealed strong stylistic clustering around specific creators
- **`country`** — Country of origin as a soft thematic signal; EDA showed strong regional production patterns
- **`rating`** — Maturity rating as a target audience signal; EDA showed rating strongly correlates with content tone

Feature weighting was achieved by string repetition before vectorization, amplifying the TF-IDF scores for high-signal fields and effectively biasing the model toward genre and director alignment — a decision directly supported by EDA findings.

### Numerical Features

| Feature | EDA Motivation | Description |
|---|---|---|
| `duration_minutes` | Duration varies significantly across content types | Movie runtime in minutes |
| `num_genres` | Multi-genre titles cluster differently | Count of assigned genre tags |
| `is_movie` | EDA revealed structural differences between types | Binary flag: 1 for Movie, 0 for TV Show |
| `release_age` | Temporal trends showed catalog skew toward recent content | Years since original release |
| `title_len` | Supplementary structural descriptor | Number of words in the title |

All numerical features were normalized using **MinMaxScaler** to bring them to a uniform [0, 1] range before computing numeric similarity. Missing numeric values were imputed with column medians to reduce distribution skew.

---

## Natural Language Processing (NLP)

**TF-IDF (Term Frequency-Inverse Document Frequency)** vectorization was applied to the composite text feature column. TF-IDF was chosen over simpler bag-of-words or one-hot encoding approaches because it:

- Penalizes extremely common tokens that carry little discriminative information
- Amplifies rare but meaningful tokens (specific director names, niche genre terms)
- Natively supports English stop-word removal

By compressing multi-token entities into single tokens before vectorization (e.g., `"Sci-Fi & Fantasy"` → `"sci-fi&fantasy"`), the vocabulary becomes more granular and avoids ambiguous partial matches. Genres and directors were pre-weighted at 3x and 2x respectively, making their corresponding TF-IDF dimensions proportionally more influential in the final feature space.

**Text similarity** is subsequently computed as the **linear kernel** of the TF-IDF matrix — equivalent to cosine similarity for L2-normalized vectors, but computationally more efficient at scale.

## Recommendation Modeling

This is an **unsupervised, similarity-based system** — no labeled training data or ground-truth preferences are required. The model relies entirely on learned content representations.

### Similarity Strategy

| Component | Method | Weight |
|---|---|---|
| Text Similarity | Linear Kernel on TF-IDF matrix | 85% |
| Numeric Similarity | Cosine Similarity on scaled numerical features | 15% |

```
hybrid_similarity = 0.85 * text_similarity + 0.15 * numeric_similarity
```

The 85/15 split was chosen empirically to prioritize thematic and categorical relevance (text) while allowing structural attributes (numeric) to serve as a tiebreaker between semantically close items.

### Fuzzy Title Matching

User inputs are rarely exact. The system uses **`difflib.get_close_matches`** to perform fuzzy string matching against the full title catalog, applying a similarity cutoff of 0.5. Matching is performed case-insensitively. This makes the system robust to partial titles, minor typos, and inconsistent casing.

## Interactive Recommendation Interface

The recommendation interface is built using **IPython widgets (`ipywidgets`)** and rendered in-notebook via **HTML**. The interface provides:

- **Real-time title search:** A text input box that triggers recommendations immediately on value change.
- **Fuzzy match disclosure:** The system displays the closest matched title so users can verify the lookup.
- **Recommendation table:** Top-N recommendations rendered as an HTML table with columns for title, content type, genres, and director.
- **Content Trend Insights:** Aggregated statistics derived from the neighborhood of similar titles, including dominant genres, frequent directors, and top countries of origin — giving users interpretable context for why recommendations were made.

## Results & Capabilities

The deployed system successfully:

- **Generates contextually accurate recommendations** by combining genre, director, and country signals with structured metadata, outperforming naive single-field lookups.
- **Handles noisy or incomplete user input** gracefully using fuzzy matching, making the system production-friendly.
- **Bridges structured and unstructured data** within a single, unified similarity framework — a key challenge in real-world recommendation systems.
- **Produces interpretable content trends** that allow users to understand the characteristics of recommended content clusters (e.g., dominant genres, prolific director patterns).
- **Scales to full catalog size** using efficient vectorized matrix operations via NumPy and Scikit-learn, without requiring model retraining per query.

### How EDA Improved Recommendation Quality

The EDA phase had a direct, measurable impact on the quality of the recommendation outputs:

- **Reduced noise:** EDA revealed which fields had high missing-value rates and limited variance, allowing them to be excluded or down-weighted. This prevents low-signal metadata from diluting recommendation accuracy.
- **Improved similarity relevance:** By identifying that genre and director are the strongest categorical differentiators in the catalog, the model was tuned to weight these fields appropriately — resulting in recommendations that share thematic and stylistic properties rather than superficial metadata overlap.
- **Prevented over-reliance on any single field:** EDA showed that no single column is sufficient to characterize content (e.g., ratings alone span many genres). The multi-field hybrid approach was a direct response to this finding, ensuring similarity is computed across a representative feature set.
